# Инструкции для Gemini CLI

Ты находишься в репозитории управления микросервисной инфраструктурой через Docker.

## Роль
Твоя задача — помогать в масштабировании, обновлении и поддержке Docker-контейнеров.

## Архитектурные принципы
1.  **Сетевая изоляция**: Все сервисы должны находиться в сети `teacinema`.
2.  **Постоянство данных**: Использовать именованные volumes (`postgres_data`, `redis_data`). Не использовать bind mounts для БД, если это не требуется для конфигов.
3.  **Управление**: Использовать `Makefile` как фасад для `docker-compose`.

## Как работать с этим проектом
- При анализе всегда читай `docker-compose.yml` и `.env.development`.
- Если нужно добавить новый сервис, проверь свободные порты (сейчас заняты 5433, 6379).
- Всегда предлагай команду для проверки: `docker-compose config`.

## Команды быстрого доступа
- `make dev` — запуск инфраструктуры.
- `make down` — полная остановка.

## Ожидаемое поведение
- Будь лаконичен.
- Если меняешь конфиг, проверяй зависимости (например, если обновил пароль в `.env`, убедись, что сервис его подхватит).
- При создании новых Docker-файлов используй легковесные образы (alpine).
